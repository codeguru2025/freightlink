### Authentication Rules

* The app uses **Google OAuth only**
* No email/password auth
* Google OAuth Client ID and Client Secret are already stored in environment **secrets**

  * Authenticate and restore session

* No secrets hardcoded — read from environment variables only

## WALLET SYSTEM (CRITICAL)

### Transporter Wallet

* Holds balance for commissions
* Can be topped up via Paynow
* Balance is checked BEFORE bid placement

### Commission Logic (MANDATORY)

```
commission = tonnes × distance_km × commission_rate_per_tonne_km
```

## BID PLACEMENT RULES (IMPORTANT CHANGE)

🚫 **DO NOT allow bid placement if wallet balance < required commission**

### On Bid Attempt:

1. Calculate required commission
2. Check transporter wallet balance
3. If balance is insufficient:

   * Block bid placement
   * Prompt transporter to top up wallet
4. If sufficient:

   * Allow bid placement
   * Reserve (soft-lock) commission amount

This ensures:

* No failed bid acceptance
* No unpaid commissions
* Clean UX

---

## BID ACCEPTANCE FLOW (SHIPPER)

1. Shipper sees:

   * Anonymous transporter IDs
   * Bid amounts
   * Ratings only
2. Shipper selects ONE bid
3. On acceptance:

   * Bid is locked
   * All other bids auto-expire
   * Commission is deducted from transporter wallet
   * Transporter full profile is revealed

## DUPLICATE ACCEPTANCE PREVENTION

To prevent double acceptance:

* Use atomic transaction / database lock
* Consignment status changes from:

  ```
  OPEN → ASSIGNED
  ```
* Once ASSIGNED:

  * No further bids allowed
  * No further acceptance allowed
* Bid acceptance must be **idempotent**

### Transporter Sees:
